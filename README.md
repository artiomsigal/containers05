# Лабораторная работа №5: Запуск сайта в контейнере
## Цель работы
Изучить процесс создания Docker-образа для веб-приложения с использованием Apache, PHP и MariaDB, а также развернуть сайт WordPress в контейнере.
## Задание
Создать Dockerfile для сборки контейнера с Apache HTTP Server, PHP (mod_php) и MariaDB. Настроить конфигурационные файлы, установить WordPress и проверить его работоспособность.
## Описание выполнения работы
1. Создан репозиторий containers05 и склонирован на компьютер.
2. Создана структура проекта:
Dockerfile
папка files с конфигурациями:
- apache2
- php
- mariadb
3. Создан Dockerfile, в котором:
- установлен Apache, PHP, MariaDB
- добавлен WordPress
- подключены конфигурационные файлы
- настроен запуск через supervisor
4. Собран Docker-образ:
`docker build -t apache2-php-mariadb .`
5. Запущен контейнер:
`docker run -d -p 8000:80 --name apache2-php-mariadb apache2-php-mariadb`
6. Из контейнера извлечены конфигурационные файлы и отредактированы:
- apache2
- php
- mariadb
7. Выполнена настройка:
- ServerName
- DirectoryIndex
- параметры PHP
- логирование MariaDB
8. Установлен WordPress через браузер:
`http://localhost:8000/wordpress`
9. Создан файл `wp-config.php` и подключена база данных.
10. Проверена работоспособность сайта.
## Ответы на вопросы
1. Какие файлы конфигурации были изменены?
- 000-default.conf (Apache)
- apache2.conf
- php.ini
- 50-server.cnf (MariaDB)
- wp-config.php
2. За что отвечает инструкция DirectoryIndex?
Инструкция `DirectoryIndex` определяет, какой файл будет открываться по умолчанию при обращении к каталогу.
Например:
`DirectoryIndex index.php index.html`
означает, что сначала будет загружаться `index.php`, затем `index.html`.
3. Зачем нужен файл wp-config.php?
Файл `wp-config.php` содержит настройки подключения WordPress к базе данных, а также основные параметры работы сайта (логин, пароль, имя базы, хост и др.). Без него WordPress не сможет работать.
4. За что отвечает параметр post_max_size?
Параметр `post_max_size` определяет максимальный размер данных, отправляемых методом POST (например, при загрузке файлов через формы).
5. Недостатки созданного образа контейнера
- Отсутствие автоматической инициализации базы данных
- Все сервисы (Apache и MariaDB) работают в одном контейнере (не по best practice)
- Нет разделения на несколько контейнеров (docker-compose)
- Нет постоянного хранения данных без явного подключения тома
- Установка лишних пакетов увеличивает размер образа
